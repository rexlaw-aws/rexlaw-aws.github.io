    已知限制和服務配額¶

    Amazon EKS 可用於各種工作負載,並可與廣泛的 AWS 服務互動,我們已看到客戶工作負載遇到類似範圍的 AWS 服務配額和其他問題,這些問題阻礙了可擴展性。

    您的 AWS 帳戶有預設配額(您的團隊可以請求的每個 AWS 資源的上限)。每個 AWS 服務都定義了自己的配額,並且配額通常是特定於區域的。您可以請求增加某些配額(軟限制),而其他配額則無法增加(硬限制)。在設計應用程序時,您應該考慮這些值。請定期查看這些服務限制,並在應用程序設計中納入。

    您可以在 AWS Service Quotas 控制台或使用 AWS CLI 查看您帳戶中的使用情況並提出配額增加請求。請參考相應 AWS 服務的 AWS 文檔,了解有關服務配額以及增加配額的任何進一步限制或通知的更多詳細信息。

    Note

    Amazon EKS 服務配額列出了服務配額,並提供了在可用的情況下請求增加的鏈接。

    AWS 請求節流¶

    AWS 服務還實施了請求節流,以確保它們保持高性能和可用性,以供所有客戶使用。與服務配額類似,每個 AWS 服務都維護自己的請求節流閾值。如果您的工作負載需要快速發出大量 API 調用,或者如果您在應用程序中遇到請求節流錯誤,請考慮查看相應的 AWS 服務文檔。

    在大型集群或集群急劇擴展時,與配置 EC2 網絡接口或 IP 地址相關的 EC2 API 請求可能會遇到請求節流。下表顯示了我們看到客戶遇到請求節流的一些 API 操作。 您可以在 EC2 文檔中查看 EC2 速率限制預設值和請求速率限制增加的步驟。

    變更操作 只讀操作
    AssignPrivateIpAddresses DescribeDhcpOptions
    AttachNetworkInterface DescribeInstances
    CreateNetworkInterface DescribeNetworkInterfaces
    DeleteNetworkInterface DescribeSecurityGroups
    DeleteTags DescribeTags
    DetachNetworkInterface DescribeVpcs
    ModifyNetworkInterfaceAttribute DescribeVolumes
    UnassignPrivateIpAddresses

    其他已知限制¶

    • Route 53 DNS 解析器的限制為 1024 個數據包每秒。當大型集群的 DNS 流量通過少量的 CoreDNS Pod 副本時,可能會遇到此限制。擴展 CoreDNS 並優化 DNS 行為可以避免 DNS 查找超時。

      • Route 53 還有一個相當低的速率限制,即每秒 5 個請求。如果您有大量域名需要使用 External DNS 等項目進行更新,可能會遇到速率節流和域名更新延遲。

    • 某些 Nitro 實例類型有 28 個卷附加限制,這個限制是 Amazon EBS 卷、網絡接口和 NVMe 實例存儲卷之間共享的。如果您的工作負載掛載了大量 EBS 卷,您可能會遇到使用這些實例類型實現的 Pod 密度的限制。

    • 每個 Ec2 實例可跟踪的最大連接數有限。如果您的工作負載處理大量連接,可能會遇到通信失敗或錯誤,因為已達到此最大值。您可以使用 conntrack_allowance_available 和 conntrack_allowance_exceeded 網絡性能指標來監控 EKS 工作節點上跟踪的連接數。

    • 在 EKS 環境中,etcd 存儲限制為 8 GiB ,這是根據上游指南。請監控指標 etcd_db_total_size_in_bytes 來跟踪 etcd db 大小。您可以參考警報規則 etcdBackendQuotaLowSpace 和 etcdExcessiveDatabaseGrowth 來設置此監控。
